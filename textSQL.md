# PostgreSQL Text-to-SQL Agent 设计文档

## Context

基于现有 LangChain 项目（已集成 MiniMax LLM）和 `db_utils.py` 中的 PostgreSQL 连接工具，开发一个 text-to-SQL agent，实现自然语言到 SQL 的智能转换。

## Agent 核心组件

### 1. Tools (src/tools/)

| Tool | 文件 | 功能 |
|------|------|------|
| schema_retriever | `schema_retriever_tool.py` | 获取数据库表结构信息 |
| sql_validator | `sql_validator_tool.py` | 校验 SQL 语法和安全性 |
| sql_executor | `sql_executor_tool.py` | 执行 SQL 查询并返回结果 |

### 2. Prompts (src/prompts/)

| Prompt | 文件 | 功能 |
|--------|------|------|
| text_to_sql | `text_to_sql_prompt.py` | 构建 text-to-SQL 系统提示 |

### 3. Agent (src/agent/)

| Agent | 文件 | 功能 |
|-------|------|------|
| text_to_sql_agent | `text_to_sql_agent.py` | 核心 Agent 实现 |

## 实现计划

### 阶段 1: 基础组件开发

1. **创建 `schema_retriever_tool.py`**
   - 封装 `MyDataSourceManager` 的表结构获取功能
   - 输出格式：JSON 描述表和字段

2. **创建 `sql_validator_tool.py`**
   - 使用 `sqlglot` 或正则验证 PostgreSQL 语法
   - 安全检查：仅允许 SELECT

3. **创建 `sql_executor_tool.py`**
   - 封装 SQL 执行逻辑
   - 错误处理和结果格式化

### 阶段 2: Prompt 和 Agent

4. **创建 `text_to_sql_prompt.py`**
   - 系统提示词设计
   - Few-shot 示例

5. **创建 `text_to_sql_agent.py`**
   - 使用 LangChain `create_sql_agent` 或自定义 ReAct agent

### 阶段 3: 集成与测试

6. **创建测试脚本 `test_text_to_sql.py`**
7. **更新 `pyproject.toml`** 添加依赖
